Traditional blockchain

The Blockchain

Blockchain is a distributed ledger technology that allows for secure, transparent, and tamper-proof transactions. It is a shared database that is maintained by a network of computers. Each block in the chain contains a set of transactions, and each block is linked to the previous block using cryptography. This makes it very difficult to tamper with the data in the blockchain, as any changes would be immediately visible to the entire network.

The concept of blockchain was first proposed in 1991 by Stuart Haber and W. Scott Stornetta. However, it wasn't until 2008 that blockchain technology was first used in practice, with the launch of Bitcoin. Bitcoin is a peer-to-peer electronic cash system that uses blockchain technology to track transactions.

Blockchain works by creating a chain of blocks, each of which contains a set of transactions. Each block is linked to the previous block using cryptography, which makes it very difficult to tamper with the data in the blockchain. When a new transaction is submitted to the network, it is added to the next block in the chain. Once a block is added to the chain, it is very difficult to remove or modify, as this would require changing all of the subsequent blocks in the chain.

Blockchain is secure because it is decentralized and tamper-proof. The data in the blockchain is not stored on any one computer, but rather on a network of computers. This makes it very difficult for anyone to hack or manipulate the data. Additionally, the data in the blockchain is encrypted, which makes it even more difficult to tamper with.

Blockchain is transparent because all transactions are recorded on the blockchain and can be viewed by anyone. This makes it very difficult for anyone to hide fraudulent or illegal activity. Additionally, the blockchain is auditable, which means that anyone can verify the authenticity of the data.

1. Hash Functions in Blockchain Technology

one of the fundamental components of blockchain technology is the use of hash functions. Hash functions are cryptographic algorithms used to generate a unique fingerprint, or hash, of a piece of data. This fingerprint serves multiple purposes, such as verifying data authenticity and detecting any changes made to the data.

In blockchain, hash functions play a crucial role in ensuring the integrity and security of the stored information. The most commonly used hash function in blockchain is SHA-256, renowned for its high level of security and resistance to cryptographic attacks. Other hash functions like RIPEMD-160 and BLAKE2b are also employed in specific blockchain applications.

By employing hash functions, blockchain technology ensures that the data stored on the blockchain is protected against unauthorized modifications. Additionally, the use of hash functions allows for efficient verification of data integrity without revealing the underlying information, contributing to the transparency and auditability of blockchain systems.

The most widely used hash function in blockchain technology is SHA-2 family, for example bitcoin using SHA2-256. SHA-2 (Secure Hash Algorithm 2) is a set of cryptographic hash functions designed by the United States National Security Agency (NSA). They are built using the Merkle–Damgård construction, from a one-way compression function itself built using the Davies–Meyer structure from a specialized block cipher.

2. Key encryption in blockchain

Another important cryptographic technique utilized in blockchain is asymmetric key encryption. Asymmetric key encryption, also known as public-key cryptography, involves the use of key pairs: a public key for encryption and a corresponding private key for decryption.

In the context of blockchain, asymmetric key encryption enables secure communication between parties without the need for a shared secret key. One widely used asymmetric key algorithm in blockchain is ECDHE-ECDSA, which utilizes elliptic curve cryptography and offers a high level of security. Additional algorithms like RSA and DHE-RSA may also be employed for specific blockchain applications.

The adoption of asymmetric key encryption in blockchain technology enhances the confidentiality and privacy of sensitive information. It allows participants to securely exchange data and messages without compromising the secrecy of their private keys. Asymmetric key encryption is a key component in ensuring the secure transmission and storage of data within blockchain systems.

Elliptic Curve Equation:

The secp256k1 curve is defined by the equation: y^2 = x^3 + 7 (mod p), where p is a large prime number that defines the finite field, and x and y are coordinates on the curve.

Prime Field Arithmetic:

All calculations in elliptic curve cryptography are performed in a finite field. In the case of secp256k1, the finite field is defined by a prime number p, which is 2^256 - 2^32 - 977.

Point Addition:

The addition of two points on the secp256k1 curve is defined by geometrically adding the coordinates of the points and taking into account the curve's equation. The result of adding two points P and Q is a third point R = P + Q.

Point Doubling:

Point doubling is a special case of point addition, where the point being added is the same as the point being doubled. Geometrically, this involves finding the tangent line to the curve at the point and finding the intersection of this line with the curve. The result is a new point that is the double of the original point.

Scalar Multiplication:

Scalar multiplication is the main operation used in elliptic curve cryptography. It involves multiplying a point on the curve by a scalar (a private key) to obtain another point on the curve. Mathematically, scalar multiplication can be performed using repeated point addition and doubling.

Base Point:

In secp256k1, a specific point called the base point (G) is defined on the curve. It has known coordinates (x, y) and is used as the starting point for scalar multiplication. The base point G generates all possible points on the curve when multiplied by different private keys.

Discrete Logarithm Problem:

The security of elliptic curve cryptography relies on the difficulty of solving the discrete logarithm problem. Given a point R on the curve, it is computationally difficult to determine the scalar K such that R = k * G, where G is the base point. This property ensures the security of the private key.

3. Signature for authentication

Digital signatures are cryptographic techniques employed in blockchain technology to provide authentication, integrity, and non-repudiation of data. Digital signatures use asymmetric key encryption to generate a unique signature that can be verified using the corresponding public key.

By utilizing digital signatures, blockchain systems can ensure the authenticity and integrity of transactions and other data stored on the blockchain. Common digital signature algorithms used in blockchain include DSA, ECDSA, and EdDSA, which provide robust security for verifying the origin and integrity of messages and transactions.

Digital signatures play a crucial role in blockchain applications such as authorizing payments in banking systems and validating transactions within a blockchain network. They provide a secure and tamper-proof method to prove the authenticity and authorship of data, ensuring trust and accountability within the blockchain ecosystem.

The most widely used digital signature in blockchain technology is ECDSA based on ECDLP "NIST published the Digital Signature Standard (DSS), like bitcoin or ethereum used today. ECDSA was proposed by a team of researchers, including Scott Vanstone, in the late 1980s and early 1990s. It was introduced as an alternative to traditional digital signature algorithms, such as the Digital Signature Algorithm (DSA), that were based on modular arithmetic. ECDSA leverages the mathematical properties of elliptic curves to provide efficient and secure digital signatures.

The development and standardization of ECDSA involved various organizations and institutions, including NIST (National Institute of Standards and Technology), which played a significant role in specifying the algorithm and its parameters. NIST published the Digital Signature Standard (DSS), which includes ECDSA as a variant, in its Special Publication 800-186

4. Final Hashing in Blockchain

In addition to the initial hashing process, blockchain technology often employs iterative hashing techniques to enhance security further. Iterative hashing involves repeatedly applying a hash function to the output of a previous hashing operation.

In Bitcoin, the final hash used is a combination of the SHA-256 and RIPEMD-160 hash functions. SHA-256 is used to hash the entire transaction, and then the resulting hash is further processed using RIPEMD-160 to generate a shorter hash known as the "transaction ID" or "hash160." This transaction ID is used for various purposes, including uniquely identifying transactions and addresses.

In Ethereum, the process of generating an address for a smart contract involves taking the rightmost 160 bits of the Keccak-256 hash of the contract's address creator's address and public key. These 160 bits are then passed through the RIPEMD-160 hash function, resulting in a final 20-byte address that is used to identify the smart contract on the Ethereum network.

1. Lattices

A lattice in ℝn, denoted by L, is generated by a set of linearly independent vectors 𝐵 = {𝐛1, 𝐛2, ..., 𝐛m}. It is the set of all integer linear combinations 𝐵𝑥 = ∑𝐛𝑖𝑥𝑖, where 𝑥 ∈ ℤm. The vectors 𝐛1, 𝐛2, ..., 𝐛m form a basis for L.

The lattice L can be generated using a matrix B, where the basis vectors are represented as column vectors.


2. Computational Lattice Problems

The unique structure of lattices allows for efficient evaluation of certain computations using linear algebra algorithms. The following problems can be solved efficiently:

a.) Basis Computation: Given a set of vectors 𝑔1, 𝑔2, ..., 𝑔𝑘 that generate the lattice L, compute a basis 𝑏1, 𝑏2, ..., 𝑏𝑚 for L.

b.) Membership Testing: Determine whether a given vector c belongs to the lattice L.


Computational lattice problems are generally challenging and are believed to resist attacks such as Shor's algorithm. These problems are of interest for post-quantum cryptography. Here are two such problems:

a.) Shortest Vector Problem (SVP): Given a lattice L with basis B ∈ ℝn×m and a norm ∥·∥, find a nonzero vector l ∈ L such that ∥l∥ is the shortest length among all nonzero lattice vectors.

b.) Closest Vector Problem (CVP): Given a lattice L with basis B ∈ ℝn×m and a norm ∥·∥, find a lattice vector l ∈ L that minimizes the distance ∥l - q∥ to a given vector q ∈ ℝn.

3. LWE Fundamentals:

Let ℤq = ℤ/qℤ be the ring of integers modulo q. We can create a linear equation system A · s = b, where A ∈ ℤqm×n, s ∈ ℤqn, and b ∈ ℤqm. For example, consider the following system: 

		A = [10 3 5 1; 4 1 1 2; ...; 3 1 1 5]
		b = [10 3 ... 8]

Then, the associated equations look like: 

		10 · s₁ + 3 · s₂ + 5 · s₃ + 1 · s₄ = 10
		4 · s₁ + 1 · s₂ + 1 · s₃ + 2 · s₄ = 3
		...
		3 · s₁ + 1 · s₂ + 1 · s₃ + 5 · s₄ = 8

Solving this equation system can be efficiently done using the Gaussian algorithm. However, adding even small error values e ∈ ℤqm to the equation system yields: 

		A · s + e = b

which makes solving the equation system and retrieving the solution vector s surprisingly hard. This hardness is related to the difficult lattice problems described above.


4. Decisional LWE

The LWE problem can also be posed as a decision problem, abbreviated as dLWE. Given an LWE sample (A, b) as defined above (with s and e kept secret), the task is to guess whether the values of b have been calculated as A · s + e with small error values e or whether they have been chosen arbitrarily. Both variants are equally hard. The reduction from LWE to dLWE has been proven by Regev (Regev, Lemma 4.2), and the inverse reduction from dLWE to LWE is trivial.


5. Linking LWE to Computational Lattice Problems

Consider an LWE problem of the form A · s + e = b, where A ∈ ℤqm×n, s ∈ ℤqn, and small vectors e ∈ ℤqm, b ∈ ℤqm. We can solve a concrete LWE instance by solving the closest vector problem. The closest vector to b is often the lattice vector A · s with distance e.

To understand the relationship between learning with errors and the shortest vector problem, consider the lattice L = {v ∈ ℤn+1 | (v|v′) · A = 0 mod q}, where the operator '||' denotes concatenation and A is the n×n identity matrix. It can be observed that the column vector (s, e, 1) is an element of L by verifying that (A|(-A)) · (s, e, 1) = A · s + e = b = 0 holds. It can be shown that the vector (s, e, 1) is a shortest vector in L and, therefore, a solution for the Shortest Vector Problem (SVP) for L. This means retrieving the vector (s, e, 1) directly yields the secret s and the error vector e, effectively solving the LWE system.

1. Lamport signature:

In 1979, Leslie Lamport introduced a groundbreaking method for one-time signatures. Lamport's approach involved the creation of two data sets comprising 256 random 256-bit numbers, referred to as Set A and Set B. Each of these sets formed the private key, consisting of a total of 512 values. By hashing each random number, a corresponding set of 512 hashes was derived, which constituted the public key. This article explores Lamport's method, highlights its limitations, and delves into the subsequent development of Merkle trees as a solution.

Lamport Method:

Lamport's method operates by hashing the message and then examining each bit of the resulting hash (ranging from 0 to 255). If a bit is 0, the corresponding element from Set A is used; otherwise, the corresponding element from Set B is employed. The signature itself comprises 256 random numbers selected from either Set A or Set B, while the public key consists of the 512 generated hashes. Notably, both the public and private keys measure 16KB in size, making Lamport signatures computationally demanding.

Limitations of Lamport Signatures:

Despite its innovative nature, the Lamport method suffers from a fundamental drawback. With this approach, a unique key pair is required for each individual message, which poses scalability and efficiency challenges. Recognizing this limitation, researchers sought alternatives that could overcome the need for a new key pair for every message.


2. The Emergence of Merkle Trees:

In the same year as Lamport's groundbreaking work, a solution to the key pair issue arose through the concept of Merkle trees. Merkle trees, named after Ralph Merkle, are a hashing technique widely used in various applications, including Bitcoin, Ethereum, Git distribution systems, No SQL databases, and P2P networks. By hashing and pairing data elements, Merkle trees construct a hierarchical structure, culminating in a root element known as the root hash. This root hash serves as proof of knowledge of all other hashes in the tree while keeping them concealed.

Utilizing Merkle Trees for Signatures:

The utilization of Merkle trees in the context of signatures presented a significant advancement. Instead of creating a new key pair for each message, Merkle trees allowed for the authentication of multiple messages with a single key pair. This was achieved through the concept of Many-Time Signatures (MTS) within a hypertree. By employing a hierarchical structure, MTS enabled the authentication of a large number of Few-Time Signatures (FTS) key pairs.


3. One time signature:

In the pursuit of improving the performance and reducing key sizes of signature methods, Robert Winternitz proposed the Winternitz One-Time Signature (W-OTS). Named after its creator, W-OTS allows for the simultaneous signing of multiple bits, presenting a significant advancement in the field. This article explores the principles and methodology behind W-OTS, shedding light on its key features and demonstrating its functionality through a visual illustration.

Winternitz OTS Method:

The Winternitz OTS method begins by generating 32 random numbers, each consisting of 256 bits. These 32 values constitute the private key. Subsequently, each of these values undergoes 256 iterations of hashing, resulting in the generation of 32 corresponding values that form the public key.

To create a signature using W-OTS, the message is hashed using the SHA-256 algorithm. This process yields 32 8-bit values, denoted as N1, N2, ..., N32. Each of these 8-bit values is then utilized to determine the number of iterations for subsequent hashing operations. Specifically, the value N determines the number of times the 8-bit signature value will be hashed (256 - N times).

To verify the signature, the original message is hashed using SHA-256 once again. The resulting hash is then processed by examining each 8-bit value. For each value, the 8-bit signature value is hashed a number of times equal to the corresponding message hash value (N1, N2, ...). The outcome of each operation should equal the corresponding value in the public key.


4. HORS:

Hash to Obtain Random Subset (HORS) is an advanced quantum-robust, hashed-based one-time signature scheme. By leveraging the power of hashing and indexing, HORS enables the creation of private and public key pairs to achieve secure and unique signatures. This article explores the methodology behind HORS, highlighting its key steps and demonstrating its application through a simple example.

HORS Methodology:

The HORS method begins by generating a series of integers to serve as private key values. These values, ranging from priv0 to priv255, are organized in an indexable form. Subsequently, each private key value undergoes a hashing process, resulting in a list of corresponding public key values, denoted as pub. These public key values are indexed for future reference.

In a post-quantum world, where powerful quantum computers exist, the need to eliminate or reduce reliance on state in cryptographic systems arises due to the potential impact of quantum computing on traditional cryptographic algorithms.

In a stateful digital signature scheme, the signer stores a secret key and a signing state between signatures. The signing state is used to generate the signatures, and it can be used by an attacker to forge signatures if they are able to obtain it.

In a stateless digital signature scheme, the signer does not store any state between signatures. Instead, the signing state is generated on the fly when the signer wants to create a signature. This makes it much more difficult for an attacker to forge signatures, as they would need to obtain the secret key and the random seed that was used to generate the signing state.

Stateless digital signature schemes are more efficient than stateful digital signature schemes, as they do not require the signer to store any state between signatures. They are also more secure, as they are less vulnerable to attacks.

Stateful cryptographic systems typically involve the storage of sensitive information, such as secret keys or internal states, which are used to perform cryptographic operations. These systems can be more vulnerable to attacks in a post-quantum world because quantum computers could potentially exploit the stored information to compromise the security of the system.

By eliminating or reducing state in cryptographic systems, the aim is to develop stateless algorithms that do not rely on the storage of sensitive information or the maintenance of long-term states. Stateless algorithms operate solely based on the input provided at the time of the cryptographic operation without relying on any stored information. This can potentially enhance the security of cryptographic systems against attacks that target the stored information.

Statelessness can reduce the attack surface and mitigate certain types of attacks, such as side-channel attacks that aim to extract sensitive information from the system's state. By eliminating state, cryptographic systems become more resistant to attacks that target stored secrets or exploit vulnerabilities in state management.

SWIFFT is a collection of provably secure hash functions. It is based on the concept of the fast Fourier transform (FFT). SWIFFT is not the first hash function based on FFT, but it sets itself apart by providing a mathematical proof of its security. It also uses the LLL basis reduction algorithm. By giving a security reduction to the worst-case scenario of a difficult mathematical problem, SWIFFT gives a much stronger security guarantee than most other cryptographic hash functions.

SWIFFT is a family of homomorphic hash functions provided as part of a candidate submission to the NIST hash function competition in 2008 by Vadim Lyubashevsky, Daniele Micciancio, Chris Peikert, and Alon Rosen. 1. Provably collision-resistant: it is provably computationally hard to find a binary-valued x given its image under f that is chosen randomly.

2. Universal: for any x_1,x_2 in the domain the probability that f(x_1) = f(x_2) over the choice of f in the family is the inverse of the size of the range.

3. Regular: if x is distributed uniformly then f(x) is distributed uniformly for any f in the family.

4. Randomness extractor: even if x is not distributed uniformly, f(x) is distributed uniformly over the choice of f in the family

6. Facilitates ZKPoKPs: the resulting hash value can be readily used in ZKPoKPs.

7. Facilitates post-quantum digital signatures: its homomorphism property enables short post-quantum hash-based digital signature schemes, such as K2SN-MSS.

Blockchain technology has revolutionized various industries, providing decentralized and transparent systems. Central to its security is the cryptographic algorithms that protect sensitive information, secure transactions, and maintain the immutability of the blockchain ledger. However, traditional cryptographic algorithms, such as ECC(DSA), SHA family based on Merkle-Damgård and Keccak construction, are at risk of being broken by powerful quantum computers in the future. It will impacted to everything that running or build in the top of its blockchain.

Quantum computers have the ability to exploit the weaknesses in these algorithms, rendering the security of blockchain networks compromised. This threat extends not only to financial transactions but also to sensitive data stored on the blockchain, including personal information, supply chain details, and healthcare records. The urgency to address this issue becomes apparent as we witness the rapid advancements in quantum computing technology.


The Solution: Post-Quantum Cryptography (PQC):

Post-quantum cryptography offers a promising solution to the imminent threat posed by quantum computers. PQC involves the development and implementation of new cryptographic algorithms that are resistant to attacks from both classical and quantum computers. These algorithms are designed to withstand the computational power of quantum computers, ensuring the continued security and privacy of blockchain networks.


Implementing PQC in Blockchain:

The implementation of PQC in blockchain systems requires a collaborative effort between researchers, developers, and blockchain projects. The transition to post-quantum cryptography involves careful planning and consideration to ensure compatibility, efficiency, and security.

One of the key challenges in adopting PQC is the need for consensus among blockchain stakeholders. The migration to new cryptographic algorithms requires coordination and agreement across the entire network to avoid fragmentation and potential security gaps. Additionally, blockchain projects must consider the performance implications of implementing PQC, as some post-quantum algorithms may introduce increased computational overhead.

Collaborative research and development efforts are already underway, exploring various post-quantum cryptographic algorithms suitable for blockchain applications. These include lattice-based, code-based, multivariate, and hash-based algorithms, among others. The goal is to identify algorithms that provide a balance between security, efficiency, and scalability, ensuring a seamless transition to post-quantum cryptography.
